Les symboles
Devoir chercher les caractères à chaque fois
Lors qu'on veut insérer des symboles dans notre texte (ex: "<") on ne peut pas juste mettre le caractère, parce qu'il causerait des erreures, pour plusieurs raisons : La première raison serait que le caractère est utilisé souvent dans le langage de code, alors l'éditeur de code pense que vous écrivez du code, et donc votre code ne marcherait plus, et le symbole n'apparaitera pas. Le deuxième raison serait si on utilise des accents français, puisqu'un éditeur de code et un moteur de recherche ne les reconnais pas, donc il montra d'autres symboles inconnus.
W3schools et la liste de caractères UTF-8
Premièrement, pour les symboles qui ne fonctionnent pas avec les éditeurs de code, il faut simplement utiliser leurs identifications respectives. Tous les symboles peuvent être représenté par une combinaison de lettres et chiffre entre les symboles "&" et ";" (ex: &eacute; donne "é"). Pour les accents français, on doit modifier la liste de caractères que le moteur de recherche utilise, pour faire cela, on doit inclure une ligne de commande dans notre code d'HTML : <meta charset="UTF-8" /> On inclus cette ligne de code dans la tête de notre HTML, puisqu'il n'est pas vu directement dans le site web.
Le français en codage
Le problème : Pas de fonctions en français
Lorsque vous apprenderez à coder en HTML et CSS, vous vous renderez compte que les commandes et les éléments portent des noms basé sur la langue anglaise, et il est plus difficile de comprendre si vous ne parlez pas anglais. On peut toujours donné des noms à des classes en français, mais les commandes utilisés vont toujours être utilisées en français.
Solution 1 : Coder en anglais
Lorsque vous apprenderez les langages de code, apprennez les avec une vidéo en anglais, et lorsque vous coder, utiliser des noms anglais pour vos classes. Puisque le niveau d'anglais utilisé lors du codage est très bas, il n'est pas difficile de d'apprendre, de comprendre, et de coder même si vous ne parlez pas très bien l'anglais. Pour cette raison, il est très utile d'éviter le français afin d'éviter de se mélanger.
Solution 2 : Apprendre les fonctions en anglais et utiliser les noms de classes français
Si vous avez de la difficulté d'apprendre en anglais, vous pouvez toujours suivre des tutoriels en français, et rechercher les commandes de styles si jamais vous les oubliez. Dans ce cas, il est très utile de noter les commandes qui vous apprenez au fil et leurs fonctionnalités pour des raisons d'éfficacités. Vous pouvezu toujours donner des noms en français au classes d'éléments lors que vous voudriez les cibler en CSS, et il est fortement recommandé de donner des noms selon la fonctionnalité de l'élément.
